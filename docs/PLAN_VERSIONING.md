# Plan Versioning

Plans and baselines must not be overwritten.

When a new plan is uploaded:

1. New plan is stored.
2. AI compares it with the current verified plan.
3. Added, changed and removed goals are identified.
4. Changed risks and strategies are identified.
5. Manager reviews the change summary.
6. Old plan is marked `superseded`.
7. New verified plan becomes current.
8. Historical evidence remains attached to its original goal/baseline.

Solo receives basic version history. Team receives manual comparison. Growth receives detailed AI comparison. Enterprise receives automated sync and version detection.
